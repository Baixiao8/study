导言 · INTRODUCTION

Skill 到底是个什么东西？为什么值得花半天学它？

这一节回答四个问题：是什么、为什么、给谁的、读完能干嘛。

Skill（能力包）是一组打包成「文件夹」的说明书，用来教 Claude 处理某一类具体任务或工作流程。Skill 是你能给 Claude 做的最强定制之一——与其每次会话都重新解释你的偏好、流程和领域知识，不如教它一次，往后次次都能用。

Skill 在重复性工作流上特别管用：按设计稿生成前端代码、用统一方法论做研究调查、按团队风格生成文档、编排多步骤业务流程。它能配合 Claude 自带的code execution（代码执行）和document creation（文档生成）使用。如果你已经在搭 MCP 集成，Skill 是再上一层的助攻——把「能调用工具」变成「能可靠地用对工具」。

这份指南把"构建一个有效 Skill"需要的所有事讲清楚——从规划、结构，到测试、分发。无论你是给自己写、给团队写、还是给社区写，都能在里面找到能用的模式和真实例子。

  • 想在团队内统一 Claude 使用方式的组织

两条阅读路径：如果你在做独立 Skill（不依赖 MCP），重点看「基础概念」「规划与设计」和模式中的第 1、2 类；如果你在为已有的 MCP 集成加一层智能，重点看「Skills + MCP」相关段落和模式中的第 3 类。两条路径技术规范一致，按你的场景挑相关的看。

术语速查 · GLOSSARY

看到这些英文不要慌

本指南反复出现的核心词，先扫一眼，正文里碰到能对上号。
Skill
能力包 / 技能包
一个装着说明书的文件夹，教 Claude 怎么做一类活。
SKILL.md
能力包的主说明文件
每个 Skill 必有的 Markdown 文件，文件名严格区分大小写。
YAML Frontmatter
YAML 头部元数据
SKILL.md 顶部用 --- 包起来的几行配置，告诉 Claude 这个 Skill 是什么、什么时候用。
Progressive Disclosure
渐进披露 / 按需展开
三层加载策略——只在需要时才把更多内容加载进 Claude 上下文，节省 token。
MCP (Model Context Protocol)
模型上下文协议
让 Claude 连接外部服务（Notion / Linear / Asana 等）的协议层。
Composability
可组合性
多个 Skill 可以同时启用、互不打架，你写的 Skill 要假设它不是唯一的。
Portability
可移植性
同一个 Skill 在 Claude.ai / Claude Code / API 上行为一致，写一次到处能用。
kebab-case
短横线小写命名
用短横线分词、全小写。Skill 文件夹名和 name 字段必须这种风格，例如 notion-project-setup。
Trigger Phrase
触发短语
用户可能说出来、让 Claude 决定加载这个 Skill 的关键说法。
Use Case
用例 / 使用场景
动笔前要先列出 2-3 个具体场景，Skill 是为它们写的。
skill-creator
Skill 创建助手
Anthropic 官方提供的"用来造 Skill 的 Skill"，能从自然语言描述生成完整 SKILL.md。
Connector
连接器
MCP server 在 Claude.ai 界面里的称呼，连上之后 Claude 就能调用对应服务。
