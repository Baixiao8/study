# 项目文档 · Anthropic Academy 学院全景笔记

## 项目定位

**一句话:** Anthropic Skilljar 18 门免费课的中文导览本 + 精华提炼,目的是「不上课也能拿到核心」。

**为什么做:**
- 用户想了解 AI / Claude 相关知识,但没时间逐门上 22+ 小时的视频
- Anthropic 官网只列了课程标题,没给「该不该学」的判断
- 现有的第三方博客都是堆事实(课时数、时长),没有「带判断的选课指南」
- 用户已有沉淀过的「报告」格式(marathon、skills-building-guide),这是自然的延续

**做给谁:**
- 主要用户:工程化思维、急、要 ROI、不想浪费时间逐门看视频的人
- 次要:想给团队推荐学习路径的负责人(产品 / 工程 / 教育)

## 关键决策

### 1. 不做逐字翻译,做带判断的导览

**为什么:** 用户在全局 CLAUDE.md 里明确说「反包装、反装饰、要尖锐共识」。逐字翻译没有价值——他可以自己看官网。

**怎么做:** 每门课给 5 字段精华卡:
- **讲什么** —— 一句话定位
- **给谁** —— 目标受众
- **章节** —— 模块大纲
- **我的判断** —— 值不值得花时间(尖锐共识)
- **跳过条件** —— 什么情况可以不学

### 2. 沿用 skills-building-guide 风格,不另起视觉

**为什么:** 两个都是「学习类」内容,统一视觉让仓库有系列感。
**风格关键:** newsprint 米黄纸 + Inter Tight 大标题 + 朱砂红 + JetBrains Mono 等宽。
**与 marathon 区别:** marathon 是暗色 editorial,适合「技术手册」;newsprint 适合「评论 / 导览」。

### 3. 单文件 SPA + hash 路由

**为什么:** 维护一个文件最省事,每章一个标签页,URL 可分享。
**与 marathon 区别:** marathon 是单页长文 + SVG 图表,这份是导览本(类似杂志),适合分章节切换。

### 4. 5 大轨道分组,不按课程顺序

**为什么:** 官网把 18 门课平铺,看着压力大。分轨道后,用户能立刻判断「这条轨道跟我有关」或「跳过」。
**5 大轨道:**
1. 入门基础(Claude 101 + AI Capabilities)
2. AI 素养(6 门 4D 框架系列)
3. 开发者深度(5 门 Code/Skills/Subagents/Cowork)
4. API 与协议(Claude API + MCP × 2)
5. 云平台部署(Bedrock + Vertex AI)

### 5. 核心知识精华独立成章

**为什么:** 18 门课里很多概念是反复出现的(4D 框架、Agentic Loop、Skills vs Subagents、Prompt Caching、MCP 三原语)。把这些核心概念抽出来用中文重讲一遍,**看完这章 ≈ 拿到了 18 门课最硬的知识**。

**6 个精华概念:**
1. AI Fluency 4D 框架(Delegation / Description / Discernment / Diligence)
2. Agentic Loop(Gather Context → Take Action → Verify Work)
3. Explore → Plan → Code → Commit 工作流
4. Skills vs Subagents vs Hooks vs MCP(四个概念怎么区分)
5. Prompt Caching 五条规则 + Extended Thinking 实战
6. Agent Workflow 四种模式 + MCP 三大原语

## 内容铁律

- **不写翻译腔**:不用「在...中」「由...所」「...的过程是」这类句式
- **不装饰英文**:不写 ELEVATOR PITCH / FINAL THOUGHT 这种英文小标题
- **不装饰比喻**:不用「细胞剧场」「差异化修复」这种直译科普话语
- **保留经典术语**:Skill / SKILL.md / Subagent / Hook / MCP / Agentic Loop / Prompt Caching 等专有术语保留英文,首次给中文解释
- **每门课给「我的判断」**:不光列内容,要敢说「值不值得花时间」「适合谁跳过」
